Introduction to AI Agents

Agents are LLMs that can take actions — calling tools, running code, browsing the web. Learn the ReAct loop, tool design principles, and build your first agent from scratch.

What makes something an agent

An agent is not just "an LLM with a fancy name." It is a loop that can reason, take an action, inspect the result, and decide what to do next. That extra loop is what lets it search, call tools, write code, browse, or continue a workflow instead of giving a one-shot answer.

  • Think — interpret the goal and choose the next step
  • Act — call a tool, fetch data, run code, or ask a follow-up
  • Observe — read the result from the environment
  • Repeat — continue until it can answer or safely stop
ℹ️ Workflows beat agents more often than people think

If your process is fixed and predictable, a workflow with clear steps is usually cheaper and easier to debug than a free-form agent. Use agents when the path itself needs reasoning.

1. When to use an agent vs a workflow

  • Use a workflow for forms, known pipelines, and deterministic automation
  • Use an agent for research, diagnosis, tool routing, or problems with many possible paths
  • Use a hybrid when most of the flow is fixed but one or two steps need model judgment

This distinction saves money, reduces failure modes, and makes debugging much easier later.

2. The core pieces of every production agent

Most real agents share the same building blocks:

  • LLM — chooses what to do next
  • Tools — search, compute, retrieve, call APIs, update systems
  • Memory — conversation state, scratchpad, or persistent user context
  • Guardrails — limits, approvals, and safety boundaries
  • Observability — logs, traces, and token usage

The dedicated lesson on Agent Components goes deeper, but you should already start thinking in this shape.

3. A minimal ReAct agent from scratch

The ReAct pattern is a clean mental model for first agents: reason, act, observe. The code below is intentionally small so you can see the loop, not hide it behind a framework.

Python - minimal ReAct agent
from openai import OpenAI

client = OpenAI()

SYSTEM = """You are an agent. Think step by step.
Use the format:
Thought: ...
Action: search[query] or finish[answer]
"""

def search(query):
    return f"Mock result for: {query}"

messages = [{"role": "system", "content": SYSTEM}]
question = "What is retrieval augmented generation?"
messages.append({"role": "user", "content": question})

for step in range(4):
    resp = client.chat.completions.create(model="gpt-4o-mini", messages=messages)
    text = resp.choices[0].message.content
    messages.append({"role": "assistant", "content": text})

    if "finish[" in text:
        print(text)
        break

    if "Action: search[" in text:
        query = text.split("Action: search[")[1].split("]")[0]
        observation = search(query)
        messages.append({"role": "user", "content": f"Observation: {observation}"})
💡 First-agent rule

Keep the first version observable and fake-friendly. Mock tools are fine. Hidden complexity is not. You want to understand the loop before you optimize it.

4. Failure modes to expect immediately

  1. Looping forever — no max steps or no stopping rule
  2. Hallucinated observations — the model invents tool results instead of waiting
  3. Bad tool choices — weak tool descriptions or weak system prompt
  4. No audit trail — you cannot tell why it made a decision

The paired debugging routes you will use a lot are debug lab and memory debugging.
