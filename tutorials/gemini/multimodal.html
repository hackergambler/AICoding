Gemini Multimodal — Images, Audio & Video

Learn how to build reliable Gemini multimodal flows for images, documents, and mixed prompts without turning the model into a black box.

Objective & multimodal mental model

Multimodal prompting is not just “attach an image and ask a question.” Production multimodal systems need a clear contract for what the model should look at, what format the answer must follow, and how the app should recover when the visual evidence is weak.

  • Images are good for classification, OCR-like extraction, layout understanding, and scene reasoning.
  • PDFs / screenshots are good for document review, invoice extraction, and policy comparison.
  • Mixed prompts work best when the text explains exactly which fields or visual regions matter.
Python — Gemini image workflow with structured extraction
from google import genai
from google.genai import types
from PIL import Image
import os

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])
invoice = Image.open("invoice.png")

schema = types.Schema(
    type=types.Type.OBJECT,
    properties={
        "vendor": types.Schema(type=types.Type.STRING),
        "invoice_number": types.Schema(type=types.Type.STRING),
        "currency": types.Schema(type=types.Type.STRING),
        "line_total": types.Schema(type=types.Type.NUMBER),
        "confidence_note": types.Schema(type=types.Type.STRING)
    },
    required=["vendor", "invoice_number", "currency", "line_total", "confidence_note"]
)

response = client.models.generate_content(
    model="gemini-1.5-pro",
    contents=[
        "Extract the invoice header fields. If a value is unclear, state that explicitly in confidence_note.",
        invoice,
    ],
    config=types.GenerateContentConfig(
        response_mime_type="application/json",
        response_schema=schema,
        temperature=0.1
    )
)

print(response.text)

How to prompt multimodal systems well

  1. Name the task precisely. “Summarize this image” is weak. “Extract vendor, invoice number, currency, and total” is much stronger.
  2. Describe the evidence source. Tell the model whether the answer should come from the image, the caption, the PDF text layer, or all of them.
  3. Require uncertainty reporting. A confidence note or evidence note is often more useful than forced certainty.
  4. Post-validate structured fields. Currency, dates, IDs, and totals should still be checked with deterministic code.
Python — PDF + question prompt for contract review
from google import genai
from google.genai import types
import os

client = genai.Client(api_key=os.environ["GEMINI_API_KEY"])
pdf_bytes = open("msa.pdf", "rb").read()

response = client.models.generate_content(
    model="gemini-1.5-pro",
    contents=[
        types.Part.from_bytes(data=pdf_bytes, mime_type="application/pdf"),
        "Find the renewal clause, payment terms, and termination notice period. Quote the exact section titles if possible."
    ],
    config=types.GenerateContentConfig(temperature=0.1)
)

print(response.text)
💡 Mixed-input discipline

When image understanding matters, ask the model to report what evidence it relied on. That gives your UI a way to show whether the answer came from visible text, layout interpretation, or broader reasoning.

Failure patterns to expect

  • Blurry or cropped inputs: the model invents details because the prompt did not allow uncertainty.
  • Hidden assumptions: teams ask for totals, dates, or IDs without specifying formatting or validation rules.
  • No fallback path: when the image is unreadable, the app should route to manual review or OCR escalation.
  • Massive inputs: large PDFs and many images increase latency. Chunk documents or pre-select relevant pages before calling the model.

Hands-on exercise

Create a receipt review assistant. It should accept an image or PDF receipt, extract merchant, purchase date, category, and amount, then return a JSON object and a short evidence note. Add two safeguards: one for missing totals and one for unreadable dates.

Real-world use case

Operations teams often need to process invoices, shipping labels, screenshots, contracts, and support attachments. A good Gemini multimodal workflow lets the model interpret the visual artifact, but the application still owns validation, escalation, and auditability. That is what turns a clever demo into a dependable document pipeline.
