Gemini Safety Settings and Structured Output

Balance safety categories, schema-driven generation, and application validation when building Gemini workflows for production.

In this lesson
  • Title & Objective
  • Concept Explanation
  • Code Example
  • Step-by-step breakdown
  • Hands-on Exercise
  • Real-world use case
  • Difficulty + Next Steps

1. Title & Objective

The objective is to make Gemini outputs both safe and machine-consumable. Many teams treat safety settings and structured generation as separate problems; in production they belong in the same loop.

2. Concept Explanation

Safe generation is not just about blocking content. It is about choosing thresholds that fit the use case, catching edge cases the model should not answer, and ensuring allowed outputs still match a clean schema. For enterprise flows, rejected outputs should degrade gracefully instead of silently failing.

3. Code Example

Python · Gemini schema + safety thresholds
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel(
    model_name="gemini-1.5-pro",
    safety_settings={
        "HARM_CATEGORY_HARASSMENT": "BLOCK_MEDIUM_AND_ABOVE",
        "HARM_CATEGORY_DANGEROUS_CONTENT": "BLOCK_MEDIUM_AND_ABOVE",
    },
)

prompt = "Return JSON with keys sentiment, urgency, escalation_reason, reply_outline for this customer message: 'Your billing flow is broken and I want help now.'"
resp = model.generate_content(prompt)
print(resp.text)

4. Step-by-step breakdown

  1. Choose safety thresholds based on risk, not guesswork.
  2. Define the output contract so allowed generations remain useful.
  3. Validate the JSON locally and route invalid responses to repair or review.
  4. Log blocked prompts separately from malformed outputs; they are different failure classes.
💡 Product insight

Teams often over-block early because safety feels safer than nuance. In practice, strong escalation and auditing usually beat blanket suppression for many business workflows.

5. Hands-on Exercise

Design a support-triage flow with Gemini that produces urgency, category, allowed_reply, and handoff_required. Test on benign, angry, and policy-sensitive inputs. Document which cases are blocked, which are escalated, and which pass automatically.

6. Real-world use case

This pattern is useful in internal copilots, support assistants, marketplace moderation, and any system where users may submit risky content but the business still needs an operational decision.
