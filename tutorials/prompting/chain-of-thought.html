Chain-of-Thought Prompting

Use chain-of-thought reasoning to dramatically improve LLM accuracy on math, logic, and multi-step problems.

Why Chain-of-Thought Works

LLMs predict tokens one at a time. When you force the model to reason step-by-step before answering, each reasoning step conditions the next — producing far more accurate results than asking for a direct answer. This is especially powerful for math, logic, and multi-step planning.

ℹ️ Research result

Wei et al. (2022) showed CoT prompting improves accuracy on math word problems by 40-50% on large models. The improvement is most pronounced on problems requiring multiple reasoning steps.

Zero-Shot CoT: The Magic Phrase

Python - zero-shot chain of thought
# Without CoT: LLM guesses directly (often wrong)
wrong = ask("You are a math tutor.",
    "Roger has 5 balls. He buys 2 more cans with 3 balls each. How many balls?")
print(wrong)  # May say 11 (correct) or wrong answer

# With CoT magic phrase: forces step-by-step reasoning
correct = ask("You are a math tutor.",
    "Roger has 5 balls. He buys 2 more cans with 3 balls each. How many balls? Let's think step by step.")
print(correct)  # Consistently correct with reasoning shown

Structured CoT with XML Tags

Python - structured reasoning then answer
COT_SYSTEM = """Solve problems step by step.

ALWAYS use this format:

[your reasoning here, can be multiple steps]


[final answer ONLY, no reasoning]
"""

import re

def solve_with_cot(problem: str) -> dict:
    raw = ask(COT_SYSTEM, problem)
    thinking = re.search(r"(.*?)", raw, re.DOTALL)
    answer = re.search(r"(.*?)", raw, re.DOTALL)
    return {
        "thinking": thinking.group(1).strip() if thinking else "",
        "answer": answer.group(1).strip() if answer else raw
    }

result = solve_with_cot(
    "A bat and a ball cost $1.10. The bat costs $1 more than the ball. How much does the ball cost?"
)
print(f"Thinking: {result['thinking']}")
print(f"Answer: {result['answer']}")  # $0.05 (not $0.10!)

CoT for Code Generation

Python - make the LLM plan before coding
CODING_COT = """Before writing any code:
1. Restate the problem in your own words
2. List the edge cases you will handle
3. Outline your approach in pseudocode
4. THEN write the final implementation

This produces better, more complete code."""

code = ask(CODING_COT,
    "Write a function that finds the second largest unique number in a list."
)
print(code)
💡 When NOT to use CoT

CoT increases output tokens which increases cost and latency. For simple tasks (classification, extraction, Q&A with known answers), direct prompting is faster and cheaper. Reserve CoT for complex reasoning tasks where accuracy matters more than speed.
