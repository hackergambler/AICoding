Getting Started with OpenAI API

Learn how to create an OpenAI account, generate API keys, set up your development environment, and make your very first API call. This is the foundation for everything else you'll learn.

📑 In This Lesson
  • 1. The OpenAI API mental model
  • 2. Create your account and project
  • 3. Set up a clean local environment
  • 4. Make your first successful call
  • 5. Read the response object like an engineer
  • 6. Choose a sensible starter model
  • 7. Fix common beginner failures
  • 8. Build the next layer intentionally

1. The OpenAI API mental model

Before you write code, keep the flow simple in your head: your app sends a request, OpenAI processes it with the model you choose, and you get back text, structured output, tool calls, or embeddings. Almost every later lesson is just a deeper version of this same cycle.

  • Client — your Python, JavaScript, or backend code
  • Model — the engine that generates text or decides tool calls
  • Messages or input — the instructions and context you send
  • Response object — content, tool calls, usage, and metadata
ℹ️ Good beginner path

Start with one reliable model, one short prompt, and one success case. Do not begin with streaming, tools, memory, and a web UI all at once. Production systems are built by stacking small known-good steps.

2. Create your account and project

Create an account on the OpenAI platform, verify it, and make sure you are working inside the correct project. In practice, teams often create separate projects or keys for development, staging, and production so usage is easier to track and keys can be rotated safely.

  1. Create the account and finish verification
  2. Open the API dashboard and choose the right project
  3. Create a descriptive key name such as academy-local-dev
  4. Copy it once and store it in your password manager or secret store
⚠️ Never hardcode keys

Do not paste API keys directly into source files, screenshots, or public repos. If a key is exposed, revoke it and create a new one immediately. The dedicated API keys debugging guide is the first place to go if authentication fails.

3. Set up a clean local environment

A clean environment solves most beginner problems before they happen. Use a virtual environment, install the SDK, and load secrets from the environment instead of the code.

bash - create a clean Python environment
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows PowerShell
.\.venv\Scripts\Activate.ps1

pip install --upgrade pip openai python-dotenv
.env - local development only
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
Python - load config safely
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
model = os.getenv("OPENAI_MODEL", "gpt-4o-mini")

4. Make your first successful call

Your first goal is not a fancy app. It is a request that succeeds consistently. Keep the prompt short, print the reply, and print token usage so you begin building cost awareness from day one.

Python - your first reliable chat request
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "You are a concise programming tutor."},
        {"role": "user", "content": "Explain what a Python list comprehension is in 3 bullet points."}
    ],
    temperature=0.2
)

print(response.choices[0].message.content)
print(response.usage)
curl - useful when debugging SDK issues
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {"role": "system", "content": "You are a concise assistant."},
      {"role": "user", "content": "Say hello in one sentence."}
    ]
  }'

5. Read the response object like an engineer

Beginners often only print the assistant text. That is useful, but the rest of the response tells you what happened and how much it cost.

  • response.choices[0].message.content — the actual answer
  • response.model — confirms which model served the request
  • response.usage.prompt_tokens — how much input you sent
  • response.usage.completion_tokens — how much output you paid for
  • response.usage.total_tokens — total billable token count
  • finish_reason — useful later for tools, length limits, or truncation
💡 Print usage from day one

The fastest way to control cost is to measure it. During development, log model name, total tokens, latency, and whether the request succeeded. That one habit makes later optimization much easier.

6. Choose a sensible starter model

You do not need the most powerful model to learn the API. Start with a fast, lower-cost model for tutorials and only move up when the task genuinely needs more reasoning depth, bigger context, or better quality.

Python - switch models with one variable
MODEL = "gpt-4o-mini"  # good default for tutorials and prototypes

response = client.chat.completions.create(
    model=MODEL,
    messages=[{"role": "user", "content": "Summarize this Python function."}]
)

Later, the Tokens and Models lesson helps you choose more intentionally. For now, optimize for fast feedback and stable experimentation.

7. Fix the five failures beginners hit most

  1. 401 / authentication errors — wrong key, wrong environment variable, or expired secret. Use API key triage.
  2. Connection or timeout issues — local network restrictions, corporate firewall, or unstable connection. Use network debugging.
  3. Package import problems — mixed virtual environments or old SDK versions. Use configuration debugging.
  4. Unexpected cost — prompts too long, chat history too large, or repeated retries without limits.
  5. Confusing outputs — weak system prompt or vague user request. The next lesson on chat completions fixes this.
ℹ️ Beginner debugging rule

When something breaks, change one variable at a time: first the key, then the environment, then the request body, then the model. Randomly changing everything at once hides the real cause.

8. Build the next layer intentionally

Once your first request works, your strongest next step is not "build a giant app." It is to add one new capability at a time:

  • Chat Completions Deep Dive — understand messages and roles properly
  • Tokens and Models — control cost and choose the right model
  • Function Calling — connect the model to tools and code
  • Streaming Responses — improve UX for longer outputs
  • Personal Assistant project — use the basics in a real build
