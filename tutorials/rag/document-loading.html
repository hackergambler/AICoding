Document Loading for RAG

Load documents for RAG with the right parser, preserve metadata, normalize content, and debug ingestion before retrieval quality collapses.

1. Document loading is where retrieval quality starts

RAG systems often fail before they ever reach the vector database. If the document loader loses structure, drops metadata, mangles tables, or ingests noisy boilerplate, the retrieval layer will struggle no matter how good the embedding model is.

Your first goal is not “load the file somehow.” It is to extract text in a form that still makes sense after chunking.

First principle

Ingestion quality beats retrieval cleverness. If the source text is broken, the answer layer inherits the damage.

2. Choose the loader by document type

  • PDFs: start with a fast text loader, then escalate to richer parsing when layout matters.
  • Markdown and plain text: preserve headings and file paths.
  • HTML: strip navigation chrome and repeated boilerplate.
  • Directories: load in batches but tag every chunk with source metadata.
Python — load a directory of PDFs
from langchain_community.document_loaders import DirectoryLoader, PyPDFLoader

loader = DirectoryLoader(
    "./docs",
    glob="**/*.pdf",
    loader_cls=PyPDFLoader,
)

documents = loader.load()
print(len(documents))
print(documents[0].metadata)

3. Preserve metadata so retrieved chunks stay explainable

Every loaded document should carry enough metadata to answer three questions later:

  1. Where did this text come from?
  2. What part of the source was it?
  3. Can I cite or re-open the exact origin?
Python — normalize document metadata
def normalize_document(doc, source_type: str):
    doc.metadata["source_type"] = source_type
    doc.metadata["source_path"] = doc.metadata.get("source")
    doc.metadata["page"] = doc.metadata.get("page")
    return doc

Without metadata, retrieval might still look good in a demo, but the system becomes much harder to debug and trust in production.

4. Clean and normalize before chunking

Raw document text often contains repeated whitespace, broken line wraps, headers, footers, or irrelevant boilerplate. Clean that early so chunking and embeddings operate on usable content.

Python — simple cleanup pass
import re

def clean_text(text: str) -> str:
    text = re.sub(r"\s+", " ", text)
    text = text.replace("Page 1 of 10", "")
    return text.strip()

Do not over-clean. Removing every line break can destroy structure. The aim is to remove noise while preserving meaning.

5. Build an ingestion pipeline you can inspect

A healthy RAG ingestion pipeline is observable at each stage:

  • raw files discovered,
  • documents loaded,
  • documents cleaned,
  • chunks created,
  • vectors stored,
  • retrieval verified.
Python — ingestion outline
loaded_docs = loader.load()
cleaned_docs = []

for doc in loaded_docs:
    doc.page_content = clean_text(doc.page_content)
    cleaned_docs.append(normalize_document(doc, source_type="pdf"))

print(f"Loaded {len(cleaned_docs)} cleaned documents")
What to inspect manually

Open five random chunks after ingestion. If a human would not trust those chunks as standalone evidence, your retrieval system probably should not either.

6. Common ingestion failures and next routes

  • Scanned PDFs return garbage: you may need OCR or a richer unstructured pipeline.
  • Chunks contain duplicate headers and footers: clean before splitting.
  • Metadata disappears: normalize documents into a consistent internal schema.
  • RAG answers cite the wrong source: chunk IDs or page metadata are not stable enough.
  • Continue into chunking and text splitting
  • Evaluate whether retrieval is actually helping
  • Fix ingestion and data-quality issues
  • Debug vector-store insertion and search problems
  • Use retrieval in a real research workflow
