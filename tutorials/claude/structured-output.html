Structured Output and JSON Contracts with Claude

Create contract-first Claude workflows with typed schemas, tool envelopes, and repair loops for extraction and routing systems.

In this lesson
  • Title & Objective
  • Concept Explanation
  • Code Example
  • Step-by-step breakdown
  • Hands-on Exercise

  • Difficulty + Next Steps

1. Title & Objective

The objective is to make Claude output predictable enough for pipelines that depend on exact fields, such as review moderation, document extraction, lead scoring, or research synthesis.

2. Concept Explanation

Claude is especially strong when you give it a clear role, explicit field descriptions, and narrow output expectations. Still, production systems should never trust raw model text. Use a typed contract, validate locally, and fall back to a repair prompt or human review for edge cases.

3. Code Example

Python · Claude JSON envelope
import json
from anthropic import Anthropic

client = Anthropic()

prompt = """Extract an editorial review record.
Return only JSON with keys: verdict, risk_level, claims, requires_editor.
verdict must be approve|revise|reject.
risk_level must be low|medium|high.
claims must be an array of short strings.
requires_editor must be true or false.
"""

resp = client.messages.create(
    model="claude-3-5-sonnet-latest",
    max_tokens=500,
    messages=[{"role": "user", "content": prompt + "

Article: The company says its battery lasts 10x longer."}]
)

raw = resp.content[0].text
payload = json.loads(raw)

allowed_verdicts = {"approve", "revise", "reject"}
if payload["verdict"] not in allowed_verdicts:
    raise ValueError("Invalid verdict")

4. Step-by-step breakdown

  1. Describe each field in business language, not abstract schema language only.
  2. Keep the output narrow enough that one model pass can satisfy it cleanly.
  3. Validate the enum values, booleans, and list sizes locally.
  4. Repair only malformed fields instead of regenerating the entire record every time.
💡 Reliable pattern

For editorial or legal review pipelines, store both the validated payload and the original reasoning text separately. The typed record drives the product; the long-form note supports reviewers.

5. Hands-on Exercise

Create a contract for vendor-risk analysis with fields for data_access, security_tier, required_controls, and review_summary. Test it on five messy procurement notes and measure validation failures.

6. Real-world use case

Claude structured output is useful for compliance review, research pipelines, content moderation, and any workflow where a reviewer needs both a machine-readable decision and a human-readable summary.

7. Difficulty + Next Steps

Difficulty: Intermediate. Combine this with prompt caching to reduce cost on repeated template-heavy tasks, then move into the Research Crew capstone.
