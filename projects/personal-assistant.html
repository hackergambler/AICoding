Project: Personal AI Assistant

Build a personal assistant with memory, retrieval, preference storage, safe tool use, and a practical daily-workflow loop.

What You Will Build

A personal assistant that remembers preferences, stores notes, retrieves relevant history, drafts helpful responses, and uses tools only when the user asks for them. The goal is not a flashy demo. The goal is a system that becomes more useful across days and weeks.

Core design principle

Memory should increase usefulness without increasing creepiness. Store what helps the user complete tasks, not everything the system can technically capture.

System Architecture

Conversation memory

Recent turns stay in the active context window so the assistant can answer naturally right now.

Preference store

Stable user preferences are written to a durable store with explicit categories such as writing style, timezone, or project stack.

Memory retrieval

Before the model answers, retrieve only the memories relevant to the current request instead of replaying an entire history.

Action layer

Allow bounded tools such as note creation, reminder drafting, or task summarization with confirmation for anything user-visible.

Python - simple memory write contract
from datetime import datetime

ALLOWED_MEMORY_TYPES = {"preference", "project_context", "task_state"}

def store_memory(user_id, memory_type, content, source):
    if memory_type not in ALLOWED_MEMORY_TYPES:
        raise ValueError("unsupported memory type")
    return {
        "user_id": user_id,
        "memory_type": memory_type,
        "content": content,
        "source": source,
        "created_at": datetime.utcnow().isoformat()
    }

Build Sequence

Milestone 1: Trustworthy memory writes

Do not let the assistant write arbitrary long-term memory on every turn. Add a clear memory policy: what kinds of facts are worth saving, what requires confirmation, and what should never be stored.

Milestone 2: Retrieval before response

Query memories using the current task, then pass only the top relevant memories into the model. This is usually more effective than stuffing the full chat history into the prompt.

Milestone 3: Controlled actions

Once memory retrieval works, add actions like note creation, task summaries, or reminder drafts. Make tool output explicit so the user sees what happened.

Python - retrieval before response
def build_context(query, memories, top_k=5):
    scored = []
    for memory in memories:
        score = similarity(query, memory["content"])
        scored.append((score, memory))
    top = [m for _, m in sorted(scored, reverse=True)[:top_k]]
    return {"query": query, "relevant_memories": top}

Where Personal Assistants Usually Fail

  • Memory spam: everything gets saved, so retrieval quality collapses.
  • Unsafe assumptions: the assistant acts on stale preferences as if they are always current.
  • Poor retrieval: the right memory exists but never gets surfaced at the right time.
  • Too much automation: the assistant takes actions the user expected to review first.
Keep this human-centered

The best personal assistants feel calm and helpful because they ask for confirmation when needed and keep memory categories understandable to the user.

Final Output Checklist

  1. The assistant stores only approved memory categories.
  2. Relevant memories are retrieved before answers, not sprayed into every prompt.
  3. User-visible actions require confirmation or create a reviewable draft.
  4. You can inspect what memory influenced a given answer.
  5. The assistant becomes more useful over time without becoming opaque or intrusive.
