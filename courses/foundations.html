How to study this track

A practical rhythm so the lessons compound instead of feeling like disconnected theory.

  • Do the first five lessons in order. They establish the Python, API, and JSON mental model that every later lesson assumes.
  • Treat NumPy, Pandas, and data cleaning as hands-on drills. Re-type the code and change the examples to your own data.
  • When you hit providers like OpenAI, Claude, and Gemini, focus on the shared patterns — message structure, retries, cost, and structured outputs.
  • Finish with logging, async, decorators, and testing so your early AI scripts stop feeling like fragile demos.

What this track gives you

Use this page as the high-level map, then open the highlighted lessons when you want the full breakdown, code, and exercises inside the academy’s lesson engine.

Python for real AI work

Learn the Python patterns that actually show up in ML notebooks, API clients, data pipelines, and agent tooling — not generic toy syntax.

APIs, JSON, and provider fluency

Build a clean understanding of REST, authentication, message payloads, structured outputs, and the difference between provider-specific syntax and universal patterns.

Math and data prep foundations

Get practical exposure to vectors, matrices, gradient descent, cleaning, normalization, visualization, and feature engineering so later ML lessons make intuitive sense.

Production habits from day one

Logging, retries, async calls, version control, testing, and scraping/data collection workflows turn a beginner stack into an actual engineering workflow.
